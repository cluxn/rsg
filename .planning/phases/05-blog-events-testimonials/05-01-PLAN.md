<objective>
Build the Express REST API for Blog posts, Events/News, and Testimonials — following the service/controller/route layered pattern established in Phase 1 (Plan 01-02). Public GET endpoints return published/active records only; all mutation endpoints (POST/PUT/DELETE) require a valid admin JWT cookie via `requireAuth`. Result: all four CONTENT requirements have working API endpoints that Plans 05-02, 05-03, and 05-04 can call.
</objective>

<task id="5.1" type="execute">
<name>Blog service: list, get by slug, create, update, delete</name>
<read_first>
- backend/src/db/connection.ts (query helper — use for all DB calls)
- backend/src/db/migrations/001_initial_schema.sql (blog_posts table schema: id, slug, title, body, meta_title, meta_description, published, published_at, created_at, updated_at)
- backend/src/services/settings.service.ts (pattern for service file structure)
</read_first>
<action>
Create `backend/src/services/blog.service.ts` with:
- `listPublishedPosts()`: `SELECT id, slug, title, meta_description, published_at, created_at FROM blog_posts WHERE published = TRUE ORDER BY published_at DESC`
- `getPostBySlug(slug: string)`: `SELECT * FROM blog_posts WHERE slug = ? AND published = TRUE LIMIT 1`
- `getAllPosts()`: `SELECT id, slug, title, published, published_at, created_at FROM blog_posts ORDER BY created_at DESC` (admin — includes drafts)
- `createPost(data: { title: string; slug: string; body: string; meta_title?: string; meta_description?: string; published?: boolean })`: `INSERT INTO blog_posts (slug, title, body, meta_title, meta_description, published, published_at) VALUES (?, ?, ?, ?, ?, ?, ?)` — set `published_at = NOW()` only when `published = TRUE`
- `updatePost(id: number, data: Partial<CreatePostData>)`: build SET clause from provided fields, include `updated_at = NOW()`, `published_at = NOW()` only when `published` transitions to TRUE
- `deletePost(id: number)`: `DELETE FROM blog_posts WHERE id = ?`
</action>
<acceptance_criteria>
- `backend/src/services/blog.service.ts` exports all 6 functions
- All queries use parameterized `query()` helper (no string interpolation of user input)
- `cd backend && npx tsc --noEmit` exits 0 after adding this file
</acceptance_criteria>
</task>

<task id="5.2" type="execute">
<name>Blog controller + router with public/protected split</name>
<read_first>
- backend/src/services/blog.service.ts (functions created in task 5.1)
- backend/src/middleware/auth.ts (requireAuth middleware)
- backend/src/controllers/settings.controller.ts (pattern for controller structure and error handling)
</read_first>
<action>
Create `backend/src/controllers/blog.controller.ts`:
- `getPublishedPosts(req, res)`: calls `listPublishedPosts()` → `res.json(posts)`
- `getPostBySlug(req, res)`: calls `getPostBySlug(req.params.slug)` → 404 if not found, else `res.json(post)`
- `getAllPostsAdmin(req, res)`: calls `getAllPosts()` → `res.json(posts)` (admin only)
- `createPost(req, res)`: validate `req.body` has `title` (string, non-empty) and `slug` (string, non-empty) via Zod — `z.object({ title: z.string().min(1), slug: z.string().min(1), body: z.string().default(''), meta_title: z.string().optional(), meta_description: z.string().optional(), published: z.boolean().default(false) })` → calls `createPost(data)` → `res.status(201).json({ ok: true, id })`
- `updatePost(req, res)`: calls `updatePost(Number(req.params.id), req.body)` → `res.json({ ok: true })`
- `deletePost(req, res)`: calls `deletePost(Number(req.params.id))` → `res.json({ ok: true })`

Create `backend/src/routes/blog.ts`:
- `GET /` → `getPublishedPosts` (public)
- `GET /:slug` → `getPostBySlug` (public)
- `GET /admin/all` → `requireAuth`, `getAllPostsAdmin`
- `POST /` → `requireAuth`, `createPost`
- `PUT /:id` → `requireAuth`, `updatePost`
- `DELETE /:id` → `requireAuth`, `deletePost`
</action>
<acceptance_criteria>
- `GET http://localhost:4000/api/blog` returns JSON array (empty or with posts)
- `POST http://localhost:4000/api/blog` without JWT cookie returns 401
- `POST /api/blog` with valid JWT and body `{"title":"Test","slug":"test","body":"<p>body</p>","published":false}` returns `{"ok":true,"id":<number>}`
- `cd backend && npx tsc --noEmit` exits 0
</acceptance_criteria>
</task>
