<action>
Create `frontend/app/(site)/HomeQuoteForm.tsx` as a `'use client'` component:

Fields (match the existing form in page.tsx exactly):
- `name` — text input, required
- `phone` — tel input, optional (but phone OR email required — zod refine)
- `email` — email input, optional
- `product_interest` — select dropdown with the same product list as the current form: "General Inquiry" + the product categories already in the current page.tsx form options
- `message` — textarea, optional

Hidden honeypot field:
```tsx
<input type="text" name="_hp" className="hidden" tabIndex={-1} autoComplete="off" />
```
Include it in the submitted payload: if the user somehow fills it (they won't), the backend rejects silently.

Submission:
- `POST ${process.env.NEXT_PUBLIC_API_URL}/api/leads` with body `{ name, phone, email, product_interest, message, source_page: 'home', _hp: '' }`
- On 201: set `submitted = true`, show inline success message: "Thank you! We'll be in touch within 24 hours." (replaces the form)
- On 429: show error "Too many requests — please try again in a few minutes."
- On other error: show "Something went wrong. Please WhatsApp us or call directly."

State:
- `submitted: boolean` — toggles success view
- `error: string | null` — error message
- `loading: boolean` — disables submit button during request

The component receives no props — it is self-contained. The glassmorphism card wrapper (the frosted glass panel, backdrop-blur, border) stays in `page.tsx`; this component renders only the form fields and submission logic inside that wrapper.

Use `react-hook-form` + zod (already installed in frontend). Zod schema:
```ts
const schema = z.object({
  name: z.string().min(1, 'Name is required'),
  phone: z.string().optional(),
  email: z.string().email('Invalid email').optional().or(z.literal('')),
  product_interest: z.string().optional(),
  message: z.string().optional(),
}).refine(d => d.phone || d.email, { message: 'Phone or email required', path: ['phone'] });
```
</action>
