The function signature:
```
export async function sendLeadNotificationEmail(lead: {
  id: number;
  name: string;
  phone?: string | null;
  email?: string | null;
  product_interest?: string | null;
  message?: string | null;
  source_page?: string | null;
  created_at?: string;
}): Promise<boolean>
```
Returns `true` if nodemailer confirms delivery, `false` otherwise (caller decides whether to update `email_sent`).

Implementation logic:
1. Read `notification_email` from settings via `getAllSettings()`. If empty or missing, return `false` immediately.
2. Read SMTP config from env: `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, `SMTP_FROM`. If any are missing, return `false` immediately.
3. Create nodemailer transporter with `nodemailer.createTransport({ host, port: Number(port), auth: { user, pass } })`.
4. Build plain-text email body:
```
New lead submitted on RSG Profile Sheets website.

Name: {lead.name}
Phone: {lead.phone ?? 'Not provided'}
Email: {lead.email ?? 'Not provided'}
Product Interest: {lead.product_interest ?? 'Not specified'}
Source Page: {lead.source_page ?? 'Unknown'}
Submitted At: {lead.created_at ?? new Date().toISOString()}

Message:
{lead.message ?? '(no message)'}
```
5. Subject: `New lead from ${lead.name} â€” ${lead.product_interest ?? 'General enquiry'}`
6. Call `transporter.sendMail({ from: SMTP_FROM, to: notification_email, subject, text: body })` â€” wrapped in try/catch, 5-second timeout via `AbortSignal` is not applicable to nodemailer; rely on nodemailer's own connection timeout (default is sufficient).
7. On success: return `true`. On any error: catch silently, return `false`.
</action>
