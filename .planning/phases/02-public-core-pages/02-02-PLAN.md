<action>
Update `frontend/components/ui/GradientHero.tsx` to add an optional `bgImage` prop (a URL string). When provided, render a `next/image` component with `fill` prop as an absolute-positioned background layer. The gradient overlay is always rendered on top via an additional `absolute inset-0` div.

Updated interface:
```ts
interface GradientHeroProps {
  children: React.ReactNode;
  className?: string;
  minHeight?: string;
  bgImage?: string; // absolute URL — when provided, image shows behind gradient overlay
}
```

When `bgImage` is provided:
- Add `relative` positioning to the section (already present)
- Render `<Image src={bgImage} alt="" fill className="object-cover" priority />` as the bottom layer (z-0)
- The existing `bg-black/10` overlay becomes `bg-navy/60` to blend the photo with the brand gradient while still showing the image
- The `gradient-premium` class stays but its opacity reduces to `opacity-80` when bgImage is set (so the photo is visible beneath)

When `bgImage` is NOT provided (gradient-only mode for About page per D-03):
- Behavior unchanged: full `gradient-premium` background, `bg-black/10` overlay

Also update `frontend/next.config.ts` to allow Unsplash images as a remote pattern:
```ts
images: {
  remotePatterns: [
    { protocol: 'https', hostname: 'images.unsplash.com' },
  ],
},
```
