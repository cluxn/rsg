<action>
Create `admin/src/pages/Seo.tsx` exporting `SeoPage` component.

Structure: two sections on a single `/seo` page (no sub-routing per D-06).

**Section 1 — Head Scripts (top)**
- Heading: "Head Scripts"
- Sub-heading: "Paste full `<script>` tag(s) from GA, GTM, FB Pixel, etc. Injected into every public page."
- Textarea bound to settings key `seo_head_scripts`. Rows=6, monospace font (`font-mono`), full width.
- Ships empty by default (read initial value from settings; empty string if key missing).

**Section 2 — Per-Page Meta (below)**
- Heading: "Page Meta"
- Sub-heading: "Override meta title and description for each public page."
- One row per page — 6 pages:
  - `{ path: '/',        label: 'Home' }`
  - `{ path: '/about',   label: 'About' }`
  - `{ path: '/contact', label: 'Contact' }`
  - `{ path: '/blog',    label: 'Blog' }`
  - `{ path: '/events',  label: 'Events' }`
  - `{ path: '/products', label: 'Products' }`
- Each row: page label, Meta Title input (`meta_title_{path}` key), Meta Description input (`meta_desc_{path}` key).
- Settings keys: `meta_title_/`, `meta_desc_/`, `meta_title_/about`, `meta_desc_/about`, etc.

**State & mutation**
- `useQuery({ queryKey: ['settings'], queryFn: () => api.get('/settings').then(r => r.data) })`
- Local state object collecting all keys: `seo_head_scripts`, `meta_title_/`, `meta_desc_/`, `meta_title_/about`, ..., `meta_title_/products`, `meta_desc_/products`.
- Single Save button at bottom: `useMutation` calling `api.put('/settings', values)`. On success: invalidate `['settings']`, show "Settings saved." for 3 seconds (same pattern as Settings.tsx).
- Error state: show "Save failed. Please try again." if mutation.isError.
- Button disabled while `mutation.isPending`.

Styling: follow Settings.tsx — `max-w-2xl p-8`, `font-heading text-2xl text-navy mb-2` for headings, `font-body` for body text, `bg-orange text-white hover:bg-orange/90` for the Save button. Separate the two sections with a horizontal rule `<hr className="border-navy/10 my-8" />`.
</action>
