<objective>
Wire the SEO settings stored by the admin into the public frontend: (1) inject the `seo_head_scripts` value from settings into the `<head>` of every public page via the root layout; (2) convert the 6 static pages from `export const metadata` to `async generateMetadata()` that reads `meta_title_{path}` and `meta_desc_{path}` from settings, falling back to current hardcoded defaults. No new backend endpoints — uses the existing `GET /settings` response already called by `frontend/lib/api.ts getSettings()`.
</objective>

<task id="6.4" type="execute">
<name>Inject seo_head_scripts in frontend/app/layout.tsx root layout</name>
<read_first>
- frontend/app/layout.tsx (current root layout — understand existing structure before modifying)
- frontend/lib/api.ts (getSettings function — already fetches all settings, reuse it)
- frontend/AGENTS.md (read before writing any Next.js code — breaking change notice)
</read_first>
<action>
Update `frontend/app/layout.tsx`:

1. Make the default export `async`
2. At the top of the component, fetch settings and extract `seo_head_scripts`:
```tsx
let headScripts = '';
try {
  const settings = await getSettings();
  headScripts = settings.seo_head_scripts ?? '';
} catch {
  // backend unreachable — no scripts injected, page still renders
}
```
3. Inside the returned JSX, add a `<head>` element (before `<body>`) containing a `<script>` rendered with `dangerouslySetInnerHTML` when `headScripts` is non-empty. Strip outer `<script>` and `</script>` wrapper tags from the stored value before injecting (the admin pastes the full tag; the `dangerouslySetInnerHTML` target is the content only):
```tsx
{headScripts && (
  <script
    dangerouslySetInnerHTML={{
      __html: headScripts.replace(/^\s*<script[^>]*>/i, '').replace(/<\/script>\s*$/i, ''),
    }}
    suppressHydrationWarning
  />
)}
```
Place this `<script>` inside an explicit `<head>` element that wraps the existing layout head content. Next.js App Router merges explicit `<head>` content with its metadata-driven output.

4. Remove the static `export const metadata` block — replace with `export async function generateMetadata()` returning the same default values (`title: 'RSG Profile Manufacturing'`, `description: '...'`). This is required because mixing static metadata export with async layout causes a conflict in Next.js App Router.

Import `getSettings` from `@/lib/api` at the top of the file.

The `/* dangerouslySetInnerHTML used intentionally — only auth-gated admin can write seo_head_scripts */` comment must appear on the same line or above the dangerouslySetInnerHTML prop.
</action>
<acceptance_criteria>
- `frontend/app/layout.tsx` default export is async
- `getSettings()` is called in the layout with a try/catch fallback to empty string
- `dangerouslySetInnerHTML` comment is present on the script element
- When `seo_head_scripts` is set to `<script>window.__TEST__=1</script>` in settings, the rendered HTML of any public page contains `window.__TEST__=1` in the `<head>` section
- When `seo_head_scripts` is empty string, no extra `<script>` tag is added to the head
- `cd frontend && npx tsc --noEmit` exits 0
</acceptance_criteria>
</task>

<action>
In `frontend/app/(site)/page.tsx`:

1. Remove `export const metadata = { ... }` (or `export const metadata: Metadata = { ... }`)
2. Replace with:
```tsx
export async function generateMetadata(): Promise<Metadata> {
  try {
    const settings = await getSettings();
    return {
      title: settings['meta_title_/'] || 'RSG Profile Manufacturing',
      description: settings['meta_desc_/'] || 'Premium quality roofing sheets, structural steel, and building materials manufacturer.',
    };
  } catch {
    return {
      title: 'RSG Profile Manufacturing',
      description: 'Premium quality roofing sheets, structural steel, and building materials manufacturer.',
    };
  }
}
```
3. Add `import type { Metadata } from 'next'` if not already present.
4. Add `import { getSettings } from '@/lib/api'` if not already present.

Default fallback strings must match the previously hardcoded metadata values exactly.
</action>
