# Plan 03-01: DB Migration, Products/Media API & Seed Data

## Objective

Add the `product_media` junction table, populate all 10 products with real descriptions and specs from the asset inventory, and build the full backend REST API for products (public read + admin CRUD) and media (upload/list/edit/delete) — including ISR revalidation on product save.

## must_haves

```yaml
truths:
  - GET /api/products/:slug returns { id, slug, name, description, specs, media: [{ id, url, alt_text, display_order }] }
  - PUT /api/products/:slug (JWT auth) updates description + specs + linked media, then calls FRONTEND_URL/api/revalidate for that product's slug
  - POST /api/media/upload (JWT auth) with missing/empty alt_text returns 400 before touching the filesystem
  - POST /api/media/upload with valid file + alt_text returns 201 with { id, url, thumbnail_url, alt_text }
  - DELETE /api/media/:id where id is linked in product_media returns 409 with { linkedProducts: [{ slug, name }] }
  - All 10 products have non-empty description and non-empty specs array after migration/seed
  - product_media table exists with FK constraints: product_id → products(id) ON DELETE CASCADE, media_id → media(id) ON DELETE RESTRICT
```

```sql
CREATE TABLE IF NOT EXISTS product_media (
  id INT AUTO_INCREMENT PRIMARY KEY,
  product_id INT NOT NULL,
  media_id INT NOT NULL,
  display_order INT DEFAULT 0,
  FOREIGN KEY (product_id) REFERENCES products(id) ON DELETE CASCADE,
  FOREIGN KEY (media_id) REFERENCES media(id) ON DELETE RESTRICT,
  UNIQUE KEY unique_product_media (product_id, media_id)
);
```

```typescript
import multer from 'multer';
export const upload = multer({
  storage: multer.memoryStorage(),
  limits: { fileSize: 10 * 1024 * 1024 }, // 10MB
  fileFilter: (_req, file, cb) => {
    if (!file.mimetype.startsWith('image/')) {

    } else {

    }
  },
});
```

### Task 4: Media API route

<read_first>
- backend/src/routes/settings.ts (pattern for route → inline handler → pool.query)
- backend/src/middleware/auth.ts (requireAuth middleware usage)
- backend/src/middleware/upload.ts (upload middleware from Task 3)
- backend/src/db/connection.ts (pool import pattern)
</read_first>

<action>
Create `backend/src/routes/media.ts` with these endpoints:

**POST /api/media/upload** (requireAuth + upload.single('file')):
- Validate `req.body.alt_text` is present and non-empty → 400 if missing
- Generate unique filename: `${Date.now()}-${Math.random().toString(36).slice(2)}`
- Use sharp to write WebP to `uploads/${filename}.webp` and thumbnail to `uploads/${filename}_thumb.webp` (400×400, cover fit)
- INSERT into media table: `filename, original_name, alt_text, mime_type, size, url=/uploads/${filename}.webp`
- Return 201 with `{ id, url, thumbnail_url: /uploads/${filename}_thumb.webp, alt_text, original_name }`

**GET /api/media** (requireAuth):
- SELECT all from media ordered by uploaded_at DESC
- Return array of `{ id, filename, original_name, alt_text, url, thumbnail_url, uploaded_at }`
- Derive thumbnail_url from url: replace `.webp` with `_thumb.webp`

**PUT /api/media/:id** (requireAuth):
- Update alt_text only (body: `{ alt_text }`)
- Validate non-empty → 400 if empty
- Return updated record

**DELETE /api/media/:id** (requireAuth):
- Check product_media for any rows with media_id = :id
- If found: JOIN with products to get slug + name → return 409 `{ error: 'Image is in use', linkedProducts: [{ slug, name }] }`
- If not found: DELETE from media table + delete files from uploads/ directory (both .webp and _thumb.webp)
- Return 200 `{ deleted: true }`

**GET /api/media/:id/usage** (requireAuth):
- SELECT products.slug, products.name FROM product_media JOIN products ON ... WHERE media_id = :id
- Return `{ used: boolean, products: [...] }`
</action>

<acceptance_criteria>
- `backend/src/routes/media.ts` exists with all 5 endpoints (upload, list, update, delete, usage)
- POST /api/media/upload without alt_text in body returns 400 `{ error: 'alt_text is required' }`
- POST /api/media/upload with valid image + alt_text writes two files to backend/uploads/ and returns 201 with url and thumbnail_url
- DELETE /api/media/:id where the media is linked in product_media returns 409 with linkedProducts array

### Task 5: Products API route

<read_first>
- backend/src/routes/media.ts (Task 4 — pattern to follow)
- backend/src/routes/revalidate.ts (existing revalidate call pattern to backend→frontend)
- backend/src/db/migrations/001_initial_schema.sql (products and product_media schema)
- .planning/research/ARCHITECTURE.md — Pattern 1 (ISR revalidation) and Pattern 2 (layered API)
</read_first>

<action>
Create `backend/src/routes/products.ts` with these endpoints:

**GET /api/products** (public, no auth):
- SELECT id, slug, name, display_order FROM products ORDER BY display_order ASC
- Return array (used by frontend generateStaticParams and admin catalog list)

**GET /api/products/:slug** (public):
- SELECT products.*, GROUP_CONCAT/JSON for linked media
- Preferred approach: separate queries (1 for product, 1 for linked media via JOIN product_media → media)
- Return `{ id, slug, name, description, specs, media: [{ id, url, alt_text, display_order }] }` ordered by product_media.display_order
- Return 404 if slug not found

**PUT /api/products/:slug** (requireAuth):
- Body: `{ description: string, specs: [{label, value}][], media_ids: number[] }` (media_ids in display order)
- UPDATE products SET description=?, specs=? WHERE slug=?
- Replace product_media: DELETE FROM product_media WHERE product_id=?; re-INSERT in order from media_ids array
- After successful update: trigger ISR revalidation
  ```typescript
  const frontendUrl = process.env.FRONTEND_URL || 'http://localhost:3000';
  await fetch(`${frontendUrl}/api/revalidate`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ secret: process.env.REVALIDATE_SECRET, path: `/products/${slug}` }),
  }).catch(() => {}); // non-blocking — don't fail save if revalidate fails
  ```
- Return updated product (same shape as GET /:slug)

**GET /api/products/:slug/media** (requireAuth):
- SELECT media.* FROM product_media JOIN media ON ... WHERE product_id = (SELECT id FROM products WHERE slug=?) ORDER BY display_order

Register all routes in `backend/src/index.ts`:
```typescript
import { productsRouter } from './routes/products';
import { mediaRouter } from './routes/media';
app.use('/api/products', productsRouter);
app.use('/api/media', mediaRouter);
```
</action>
